Cara Install Paket .Deb Di Ubuntu

Assalamu‚Äôalaikum Wr. Wb. Software atau perangkat lunak merupakan komponen utama dalam sebuah komputer. Tanpa software yang sesuai sering kali kita tidak bisa menyelesaikan pekerjaan kita dengan mudah. Dalam sebuah sistem operasi baik itu Linux, Windows, ataupun Mac OSX ketika terinstall tentu sudah termasuk dengan beberapa software aplikasi default yang telah terinstall secara otomatis bersamaan dengan sistem operasi tersebut. Akan tetapi seringkali software atau aplikasi bawaan dari masing-masing sistem operasi belum mampu untuk memenuhi kebutuhan kita, sehingga kita perlu menginstall software lain baik dari repository resmi sistem operasi tersebut ataupun sumber luar. Nah, kali ini saya akan membahas mengenai salah satu cara untuk menginstall software dari sumber luar di Ubuntu, yaitu cara install paket deb.

Metode Install Software/Aplikasi Pada Ubuntu

Bagi pemula Ubuntu yang baru saja bermigrasi dari sistem operasi lain mungkin akan merasa kebingungan untuk menginstall software aplikasi baru pada sistem operasi ini. Sebenarnya terdapat beberapa cara untuk menginstall aplikasi pada Ubuntu, antara lain :

  • Melalui packet manager seperti : Ubuntu Software Center, Synaptic Packet Manager, dll.
    Metode ini layaknya ketika anda menginstall sebuah aplikasi pada smartphone Android anda melalui Google Play Store, atau menginstall aplikasi pada perangkat Apple anda melalu AppStore. Di mana anda hanya perlu membuka packet manager yang akan anda gunakan, dan mencari software aplikasi yang ingin anda install, dan kemudian menginstallnya hanya dengan beberapa klik saja.
  • Melalui paket installer ber-ekstensi deb.
    Metode ini layaknya anda menginstall aplikasi pada Windows, dimana anda tinggal menjalankan file installer dengan ekstensi .exe atau .msi. Bedanya, pada Ubuntu, packet deb ini nantinya juga akan mengecek kelengkapan dependensi yang telah terinstall pada sistem operasi Ubuntu anda terlebih dahulu. Jika ternyata dependensi yang dibutuhkan packet deb tersebut belum lengkap atau belum terinstall, maka secara otomatis sistem akan meminta untuk menginstall dependensi tersebut pada sistem operasi Ubuntu anda, barulah setelah dependensi-nya lengkap, paket yang ingin anda install akan mulai terinstall secara otomatis.
  • Melalui source code yang dicompile secara manual.
    Metode ini merupakan salah satu metode yang cukup sulit untuk dilakukan bagi para pemula. Dengan metode ini, anda diharuskan meng-compile paket aplikasi yang masih berupa kumpulan source code mentah, biasanya file nya akan dikemas dalam arsip ber-ekstensi .tar.gz. Apabila dalam metode instalasi dengan paket ber-ekstensi deb semua dependensi yang diperlukan akan diinstall secara otomatis, lain halnya apabila kita mengcompile source code mentah untuk menginstall software aplikasi yang kita inginkan, di mana kita diharuskan menginstall semua dependensi yang diperlukan satu-per-satu.
  • Melalui aplikasi Alldeb. Metode ini merupakan metode yang masih tergolong baru, dan sangat ramah untuk pemula. Alldeb sendiri merupakan sebuah program shell script sederhana hasil karya dari teman-teman Forum Ubuntu Indonesia. Dengan Alldeb kita dapat menginstall sebuah software aplikasi yang kita inginkan tanpa harus terhubung internet dan menginstall dependesi, karna semua dependensi dari software aplikasi yang akan kita install sudah terkemas dalam satu file installer ber-ekstensi .alldeb. Untuk info lebih lengkap silahkan kunjungi alamat web ini.
